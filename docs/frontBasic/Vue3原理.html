运行时 ​

简单来说，就是代码运行过程中执行的任务

例如：Vue3项目的packages/runtime-core目录下存放的就是Vue3的运行时核心代码，其中暴露出来render、h函数

Vue在运行过程中，使用h动态创建虚拟Dom，使用render将虚拟Dom渲染到html中

  • 使用Vue模版语法。声明式语法，仅声明我们要的结果，并不处理Dom细节

2、实现简单的运行时render

项目在运行时执行render函数，动态把vNode对象转化为实际Dom代码

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
    <div id="app"></div>
</body>

<script>
    const vNode={
        type:'div',
        props:{
            class:'test'
        },
        children:'hello world'
    }

    function render(vNode){
        const ele =document.createElement(vNode.type)
        ele.className=vNode.props.class
        ele.innerHTML=vNode.children

        document.body.appendChild(ele)
    }

    render(vNode)
</script>

</html>

编译时 ​

简单来说，就是代码运行之前处理的任务

Vue中可以通过compile提前将模版编译为render函数（build打包的过程中进行），在运行期间执行render函数

所以Vue是：编译时+运行时

Vue并没有在render中直接挂载到页面Dom上（上一节，运行时的例子），而是在编译时将模板处理为render函数，在运行时挂载到页面

其实，编译时除了将模板转换为JS对象外

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
    <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
</head>
<body>
    <div id="app"></div>
</body>

<script>

    const {compile,createApp}= Vue

    // 模版
    const html=`
        <div class="test">hello world</div>
    `
    // compile将模版编译为render函数
    const renderFunc= compile(html)
    // console.log('renderFunc',renderFunc)

    // 创建app实例，挂载到#app上
    const app=createApp({
        render:renderFunc
    })
    app.mount('#app')

</script>

</html>

对比 ​

  • 纯运行时：因为不存在编译器，所以我们需要提供复杂的对象来描述Dom或者代码运行时解析模版生成这个复杂的对象
  • 纯编译时：项目运行中对Dom的操作是动态的，纯编译时无法再运行过程中动态分析最优的操作策略（svelte是一个纯编译时的框架）
  • 运行时+编译时：Vue、React均是使用这种方案

Proxy与Reflect ​

Vue3采用了Proxy实现其响应式核心

注意：只有直接操作代理对象，才能被监听到。如果操作target则不能被监听到

js
let product={
    price:10,
    count:2
  }

  let total=0

  let effect=()=>{
    total=proxyProduct.price*proxyProduct.count
    console.log('总价格',total)
  }

  effect()

  const proxyProduct=new Proxy(product,{
    set(target,key,newVal,receiver){ // receiver就是proxyProduct，是一个代理对象
      target[key]=newVal
      effect()
      return true
    },
    get(target, key, receiver) {
      return target[key]
    }
})

ref部分 ​

reactive存在的个问题：无法实现基本类型变量的响应式更新

这是因为：reactive本质是创建了Proxy，Proxy第一个参数只能是对象，所以reactive参数也只能是对象，如果传入基本类型会报错

class的get、set ​

js
class Person {
    #age = 0 // 设置私有属性存储值，避免被外部修改
    constructor(name, age) {
    
      this.#age = age
    }
    get age() { //age属性的getter
      return this.#age
    }
    set age(val) {//age属性的setter
      if (this.checkAge(val)) { // 设置前校验格式
        this.#age = val
      } else {
        console.warn('年龄不符合要求')
      }
    }
    checkAge(val) {
      return typeof val === 'number' && !(val < 0 || val > 150);
    }
}

const tom = new Person('tom', 12)
tom.age = 45
tom.age = 151 // 年龄不符合要求
console.log(tom.age) // 45

ReactiveEffect实例（与reactive一样，fn属性存的是回调函数effect）存在类RefImpl的dep属性中

class RefImpl<T> {
  public _value: T;
  // set集合存多个ReactiveEffect实例（effect回调函数）
  public dep?: Dep = undefined; 
  public readonly __v_isRef: boolean = true;
  constructor(
    value: T,
    public readonly __v__isShallow: boolean, // 是否为复杂类型
  ) {
    // 如果是复杂数据类型，则直接使用reactive函数返回一个代理对象，存在_value中
    // 如果是简单类型，则直接存在_value中
    this._value = __v__isShallow ? value : toReactive<T>(value);
  }

  get value() {
    //收集依赖
    trackRefValue(this);
    return this._value;
  }

  set value(value: T) {
    // 触发依赖
    this._value = value;
  }
}

脏变量 ​

dirty变量

  • true 代表数据过时，应该更新
  • false 代表数据无需更新

缓存 ​

只有依赖的变量更新，才会重新触发computed的回调函数，否则直接返回缓存的值

实现 ​

  • 创建计算属性 （ ComputedRefImpl 类型）

    js
    constructor(getter) {
  	// ReactiveEffect构造函数
    // 第一个参数：把计算属性的回调，存在ReactiveEffect实例的fn属性中
    // 第二个参数：存在ReactiveEffect实例的scheduler属性中(reactive改动，触发的setter时，就会触发依赖，然后执行scheduler)
    this.effect = new ReactiveEffect(getter, () => {
      if (!this._dirty) {
        // computed返回的ref被修改，数据置为脏。重新触发依赖
        this._dirty = true;
        triggerRefValue(this);
      }
    });
    this.effect.computed = this;
  }

实现

js
// runtime-core/scheduler.js

// 微任务队列是否pending
let isFlushPending: boolean = false;

// 微任务队列
const pendingPreFlushCbs: Function[] = [];

const resolvePromise = Promise.resolve() as Promise<any>; // 默认为 Promise<void>，需要断言

// eslint-disable-next-line @typescript-eslint/no-unused-vars
let currentFlushPromise: Promise<void> | null = null;

/**
 * 加入微任务队列
 * queueCb函数：cb推入pendingQueue
 * queueFlush函数：微任务未pending，使用Promise.resolve创建一个微任务执行flushJobs
 *
 * 第二次执行，cb函数推入pendingQueue后，不会创建一个微任务了
 */
export function queuePreFlushCb(cb: Function) {
  queueCb(cb, pendingPreFlushCbs);
}

function queueCb(cb: Function, pendingQueue: Function[]) {
  pendingQueue.push(cb);
  queueFlush();
}

function queueFlush() {
  if (!isFlushPending) {
    isFlushPending = true;
    currentFlushPromise = resolvePromise.then(flushJobs);
  }
}

/**
 * 宏任务执行完毕后，会自动执行微任务flushJobs
 * 执行后，重置微任务队列为 `未pending`
 * 执行微任务队列的全部函数
 */
function flushJobs() {
  isFlushPending = false;
  flushPreFlushCbs();
}

export function flushPreFlushCbs() {
  if (pendingPreFlushCbs.length) {
    // 去重拷贝一份
    const activePreFlushCbs = [...new Set(pendingPreFlushCbs)];
    pendingPreFlushCbs.length = 0;

    for (let i = 0; i < activePreFlushCbs.length; i++) {
      activePreFlushCbs[i]();
    }
  }
}

运行时 ​

在运行时

  • 渲染器（renderer）会遍历整个虚拟Dom树，并根据虚拟Dom树构建真实Dom，这个过程称为挂载（mount）

  • 当VNode变化时，对比新、旧VNode之间的区别，并应用这些变化都真实Dom上，这个过程称为更新（patch）

在Vue3中

  • h()

    负责创建虚拟Dom。虚拟Dom节点的类型：

    • Text：文本节点
    • Comment：注释节点
    • Static：静态Dom节点
    • Fragment：片段，即包含多个根节点的模板
    • ELEMENT：Dom节点
    • COMPONENT：组件
    • TELEPORT、SUSPENSE：Vue3的内置组件
    • 等...

  • render

    将虚拟Dom负责生成真实Dom（挂载、更新）

代码分布在runtime-core、runtime-dom目录下

  • runtime-core负责渲染的核心逻辑，与平台无关（例如uniapp之类的，并不是在浏览器环境渲染页面）
  • runtime-dom是浏览器端的实现
