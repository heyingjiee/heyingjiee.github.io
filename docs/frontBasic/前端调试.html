配置文件配置项目 ​

json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Launch Chrome",
            "request": "launch",
            "type": "chrome",
            "url": "http://localhost:8080/charge-attract-customer-platform", //网页服务器的地址（本地开发一般就是devServer的地址）
            "runtimeExecutable":"dev", //指定使用浏览器的版本，默认stable。可以去官网安装dev版本https://www.google.cn/intl/zh-CN/chrome/dev/
            "userDataDir": false, //false是使用浏览器默认的数据存储目录，特别注意浏览器目录同时只能被一个实例访问，设置false时，要保证浏览器必须只能打开一个。也可以手动指定存储数据的目录。注意：false使用默认目录时，浏览器中安装的插件、历史记录才会存在
            "runtimeArgs": [
                "--auto-open-devtools-for-tabs", //启动浏览器时，打开调试器
                // "--incognito" //浏览器使用无痕模式
            ],
            "sourceMaps": true,  //浏览器开启sourceMaps
            "webRoot": "${workspaceFolder}"
        },
      
    ]
}

补充

  • 推荐安装dev版本

    启动多个浏览器时，再设置使用默认数据存储目录，则会报错。必须关闭所有浏览器实例

    js
    "userDataDir": false //使用浏览器默认数据存储目录
    1

    安装Dev版本，这样可以调试时使用游览器默认数据存储目录，就不会和稳定版本冲突导致出现上面的报错。而且，只要设置使用dev的用户数据存储目录，那么在dev版本安装的插件都可以正常使用

    json
    "runtimeExecutable":"dev" //使用dev版本
    1

调试Vue ​

对于Vue项目来说，浏览器实际运行的是编译后的代码。开启sourceMaps选项后，可以看到浏览器的控制台显示的是编译前的源码码。

浏览器找到了编译前代码的位置，而VSCode呢？

VSCode自动打开了一个 login.vue?01f7的文件，因为而不是实际的login.vue文件，所以VSCode提示了不可编辑该文件

这主要是因为Vue自动生成的source map文件导致，浏览器找到的文件名后拼接了 ?hash，进而VSCode真的去尝试在本地打开这个文件

login.vue?01f7
1

如何解决？本质上来说只要我们能去掉?hash就可以了

  • vue/cli创建的项目，在vue.config.vue中配置devtool选项

    此选项控制是否生成，以及如何生成 source map。

    修改后，从新运行即可发现映射到了正确的文件名

    在增加一条，添加浏览器的源码到本地文件的映射即可

    json
    "sourceMapPathOverrides": {
   "webpack://inquire/src/*":"${workspaceFolder}/src/*" 
}
    1
    2
    3

    也可以用?:做占位符

    json
    "sourceMapPathOverrides": {
   "webpack://?:/src/*":"${workspaceFolder}/src/*" 
}
    1
    2
    3

    启动调试，可以看到成功映射到了本地的login.vue
